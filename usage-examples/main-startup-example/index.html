    main() startup example

    Example an app that starts up & loads arguments from main(String[] args)

    Full source and tests for this example can be found here and is an extension of the example on the home page.

     1package org.simple;
 2
 3import org.yarnandtail.andhow.*;
 4import org.yarnandtail.andhow.property.*;
 5
 6@GroupInfo(name="Launch Config", desc="More details...")
 7public class SimpleStringArgs {
 8  
 9 final static IntProp COUNT_DOWN_START = IntProp.builder().mustBeNonNull()
10   .desc("Start the countdown from this number")
11   .mustBeGreaterThanOrEqualTo(1).build();
12 
13 private final static StrProp LAUNCH_CMD = StrProp.builder().mustBeNonNull()
14   .desc("What to say when its time to launch")
15   .mustMatchRegex(".*Go.*").build();
16 
17 private final static BolProp NOTIFY_NEWS = BolProp.builder().defaultValue(true)
18   .desc("If true, post launch events to the AP news feed").build();
19 
20 public String launch() {
21  String launch = "";
22  
23  for (int i = COUNT_DOWN_START.getValue(); i >= 1; i--) {
24   launch = launch += i + "...";
25  }
26  
27  return launch + LAUNCH_CMD.getValue();
28 }
29 
30 public static void main(String[] args) {
31  
32  AndHow.findConfig()
33    .setCmdLineArgs(args)
34    .addFixedValue(NOTIFY_NEWS, false) //turn off for cmd line use
35    .build();
36  
37  SimpleStringArgs gs = new SimpleStringArgs();
38  System.out.println(gs.launch());
39 }
40 
41 public boolean isNewsNotified() { return NOTIFY_NEWS.getValue(); }
42}

    This example is just like the home page example with a few changes. First, we added a new Boolean property NOTIFY_NEWS. ‘Notify’ might post to a news feed to let the world know about the launch: You probably want to happen when the full enterprise system is preparing a real launch (the default value is true), you probably don’t want that to happen if a developer is just running this locally from command line.

    Next, we added a @GroupInfo annotation so we could add some notes and help information about configuring the Properties contained in this class. When AndHow generates a sample configuration file, this information will be included.

    We’ve also made two key additions to the main method:
    - setCmdLineArgs(args) injects the command line arguments into the AndHow configuration so they are included when property values are read - addFixedValue(NOTIFY_NEWS, false) forces NOTIFY_NEWS to always be false when run from the command line because we don’t local testing to show up on CNN. When a ‘real launch’ happens, this class would probably be used in larger framework and the main method not used.

    One more small change: The default values were removed from COUNT_DOWN_START and LAUNCH_CMD. When this is run for the first time, an error is thrown because these properties are required. Any time there is a startup error, AndHow generates configuration samples. Here is an excerpt - Note that the @GroupInfo information was added:

    # ##########################################################################################
# Property Group 'Launch Config' - More details...
#   Defined in org.simple.SimpleStringArgs

#

#

# The property value must match the regex expression '.*Go.*'

#

org.simple.SimpleStringArgs.NOTIFY_NEWS = true

    If you try out this sample project, the generated property file has already been copied to the resources directory as /andhow.properties - delete or rename it to force AndHow to generate another copy.
